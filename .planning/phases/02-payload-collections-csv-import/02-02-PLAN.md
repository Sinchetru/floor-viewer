# Plan 02-02: CSV Import API Routes

## Goal

Build two Next.js API route handlers — one per collection — that accept a CSV file upload, parse German column headers, validate rows, upsert into Payload, and return an import summary.

## Context

Depends on: Plan 02-01 complete (RoomGeometry and RoomData collections exist, types generated, migration run).

These are Next.js App Router route handlers at:
- `src/app/api/import/room-geometry/route.ts`
- `src/app/api/import/room-data/route.ts`

Both routes are admin-only (checked via Payload's `getPayload` + session). Both use `papaparse` (to be installed) for CSV parsing.

**German header → DB field mapping:**

RoomGeometry:
- `Raum-ID` → `room_id`
- `Pfaddaten` → `path_data`
- `Gebäude` → `building`
- `Ebene` → `floor`

RoomData:
- `Raum-ID` → `room_id`
- `Raumnummer` → `room_number`
- `Raumtyp` → `room_type`
- `Raumtyp-Code` → `room_type_code`
- `Fläche` → `area_sqm`
- `DIN277-Code` → `din277_code`
- `DIN277-Name` → `din277_name`
- `Kostenstelle` → `cost_center`
- `Kostenstellen-Name` → `cost_center_name`
- `Sondernutzung` → `special_use`
- `Eigentümer` → `owner`
- `Standort` → `location`
- `Raumnotiz` → `room_note`
- `Nutzungsnotiz` → `usage_note`
- `Nutzung-von` → `usage_from`
- `Nutzung-bis` → `usage_to`

**Upsert logic:** For each valid row, search `room-geometry` (or `room-data`) collection for existing doc with `room_id` equals the incoming value. If found, call `payload.update()`; if not, call `payload.create()`. Within a single CSV, if `Raum-ID` appears more than once, the last row wins (process rows in order, upsert each — later rows naturally overwrite earlier ones).

**Validation:** A row is invalid (skipped) if any of these are blank/missing: `Raum-ID`. For RoomData only, also skip if `Kostenstelle` or `Fläche` are blank. Import continues past invalid rows.

**Summary response:**
```json
{
  "total": 50,
  "imported": 47,
  "skipped": 3,
  "errors": [
    { "row": 12, "reason": "Missing Raum-ID" },
    { "row": 23, "reason": "Missing Kostenstelle" }
  ]
}
```

## Threat Model (ASVS L1)

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Non-admin uploads CSV and overwrites all room data | Critical | Route handler checks Payload session — reject with 403 if user is not admin role before processing any file content |
| Malicious file disguised as CSV (e.g. script, executable) | High | Accept only `text/csv` and `text/plain` MIME types; reject anything else with 400 before parsing |
| Oversized file causes memory exhaustion / DoS | High | Enforce 10 MB file size limit; return 413 if exceeded before parsing |
| CSV injection — formula injection via cell values (e.g. `=CMD(...)`) | Medium | Data is stored as plain text and rendered in admin UI — Payload's admin escapes HTML; no spreadsheet export in this phase so risk is contained. Sanitize values by stripping leading `=`, `+`, `-`, `@` characters from text fields before upsert |
| Path traversal via filename | Low | File content is parsed in memory from FormData; filename is never used to construct a filesystem path |
| Upsert overwrites a record with blank/empty fields from a partial CSV | Medium | Only upsert fields that are present in the CSV row's mapping; do not send `undefined` values to Payload (omit missing optional fields from the update payload) |
</threat_model>

## Tasks

### Task 1: Install papaparse

**File:** `package.json` (via pnpm)

**Action:** Run `pnpm add papaparse` and `pnpm add -D @types/papaparse`.

**Details:** papaparse is chosen over csv-parse because it is browser/edge-compatible (no Node stream dependency), handles UTF-8 with BOM (common in German Windows CSV exports), and has a simple synchronous string-parsing API suitable for in-memory route handlers.

---

### Task 2: Create shared CSV import utility

**File:** `src/lib/csv-import.ts`

**Action:** Create a shared utility module that both route handlers import. Export two things:

1. `GEOMETRY_HEADER_MAP` — object mapping German geometry headers to DB field names (per mapping above)
2. `ROOMDATA_HEADER_MAP` — object mapping German room data headers to DB field names (per mapping above)
3. `parseAndMapCsv(csvText: string, headerMap: Record<string, string>): Record<string, string>[]` — parses CSV text with papaparse (header: true, skipEmptyLines: true), remaps column names using `headerMap`, returns array of row objects with DB field names as keys. Columns not in the map are silently dropped.
4. `sanitizeCsvValue(value: string): string` — strips leading `=`, `+`, `-`, `@` from any string value to prevent CSV injection.

**Details:** Apply `sanitizeCsvValue` to every string field value inside `parseAndMapCsv` before returning rows. Numeric fields (`area_sqm`) will be parsed to `number` by the route handler, not here (keeps utility pure string-level).

### Task 3: Create RoomGeometry import route

**File:** `src/app/api/import/room-geometry/route.ts`

**Action:** Create a POST route handler:

```typescript
export async function POST(request: Request): Promise<Response>
```

Steps in order:
1. Get Payload instance via `import { getPayload } from 'payload'` and `import config from '@payload-config'`. Call `const payload = await getPayload({ config })`.
2. Get the current user from the request: `const { user } = await payload.auth({ headers: request.headers })`. If `!user || user.role !== 'admin'`, return `Response.json({ error: 'Forbidden' }, { status: 403 })`.
3. Parse the `multipart/form-data` body: `const formData = await request.formData()`. Get the file: `const file = formData.get('file') as File | null`. If null, return 400.
4. Check MIME type — accept only `text/csv` and `text/plain`. If wrong type, return `Response.json({ error: 'Invalid file type. Upload a CSV file.' }, { status: 400 })`.
5. Check file size — if `file.size > 10 * 1024 * 1024`, return `Response.json({ error: 'File too large. Maximum 10 MB.' }, { status: 413 })`.
6. Read file text: `const csvText = await file.text()`.
7. Call `parseAndMapCsv(csvText, GEOMETRY_HEADER_MAP)` from `src/lib/csv-import.ts`.
8. For each row: validate `room_id` is non-empty. If missing, push to `errors` array with row number and reason, skip.
9. For valid rows: query `payload.find({ collection: 'room-geometry', where: { room_id: { equals: row.room_id } }, limit: 1 })`. If `docs.length > 0`, call `payload.update({ collection: 'room-geometry', id: docs[0].id, data: { path_data: row.path_data, building: row.building, floor: row.floor } })`. If not found, call `payload.create({ collection: 'room-geometry', data: row })`.
10. Return summary JSON: `{ total, imported, skipped, errors }`.

### Task 4: Create RoomData import route

**File:** `src/app/api/import/room-data/route.ts`

**Action:** Same structure as RoomGeometry route. Differences:

- Uses `ROOMDATA_HEADER_MAP` from `src/lib/csv-import.ts`
- Collection slug: `'room-data'`
- Validation: skip row if `room_id`, `cost_center`, or `area_sqm` (mapped from `Fläche`) are blank/missing
- Before upsert, parse `area_sqm` to number: `parseFloat(row.area_sqm)` — if NaN, treat as `undefined` (omit the field, do not fail the row)
- Parse `usage_from` and `usage_to` as ISO date strings if present, otherwise omit

**Details:** The upsert update payload should only include fields that are non-`undefined` after parsing. Construct the data object by checking each field and only including keys that have a value.
