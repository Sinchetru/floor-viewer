# Phase 1 βÄî UI Design Contract

> Visual and interaction contract for the login page, portal page, and tile grid.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | none βÄî use `npx shadcn@latest init` defaults (neutral palette, rounded corners, system font) |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react (installed with shadcn by default) |
| Font | system-ui (OS default βÄî no Google Fonts, no custom font loading) |

**Initialization command (Wave 0):**
```bash
npx shadcn@latest init
```
Accept all defaults. This produces `components.json` and writes CSS variables to `src/app/(frontend)/globals.css`. Confirm `components.json` exists before proceeding to component installation.

**shadcn components required this phase:**
- `button` βÄî login form submit, future nav actions
- `card` βÄî portal tile container
- `input` βÄî email and password fields
- `label` βÄî form field labels
- `form` βÄî wraps login inputs with validation wiring

Install after init:
```bash
npx shadcn@latest add button card input label form
```

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, badge padding |
| sm | 8px | Input inner padding (vertical), label-to-input gap |
| md | 16px | Default element spacing, card inner padding, form field gaps |
| lg | 24px | Section padding, tile grid gap |
| xl | 32px | Login card internal vertical padding |
| 2xl | 48px | Page vertical centering offset |
| 3xl | 64px | Not used this phase |

Exceptions:
- Touch targets (buttons, tiles): minimum 44px height per WCAG 2.5.5

## Typography

All sizes use `font-family: system-ui` βÄî no custom typeface.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 14px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Body (14px/400): form helper text, tile descriptions, error messages
- Label (14px/500): form field labels ("E-Mail", "Passwort"), nav items
- Heading (20px/600): portal section title ("Module"), login card title ("Anmeldung")
- Display (28px/600): app name or page-level heading βÄî reserved, not rendered in Phase 1

## Color

shadcn neutral palette via CSS custom properties. Light mode only for Phase 1.

| Role | Tailwind / CSS var | Approx value | Usage |
|------|--------------------|--------------|-------|
| Dominant (60%) | `bg-background` / `--background` | #ffffff | Page background, form backgrounds |
| Secondary (30%) | `bg-muted` / `--muted` | #f5f5f5 (neutral-100) | Portal page background, tile hover state |
| Accent (10%) | `bg-primary` / `--primary` | #171717 (neutral-900) | Submit button fill, focused input ring |
| Destructive | `bg-destructive` / `--destructive` | #ef4444 (red-500) | Login error text and border only |

Accent reserved for:
1. Primary submit button fill ("Anmelden")
2. Focused input outline ring
3. Active/selected tile border indicator (future phases)

Never use accent on: informational text, decorative elements, icons without action semantics.

Out-of-scope room rendering color (Phase 3+): `bg-neutral-300` / #d4d4d4 βÄî reserved, not used this phase.

## Interaction States

### Login Form

| State | Visual |
|-------|--------|
| Default | White input, `border-input` border, placeholder in `text-muted-foreground` |
| Focused | `ring-2 ring-primary` on input (shadcn default) |
| Error (field) | `border-destructive` on input, error message below in `text-destructive`, 14px/400 |
| Loading (submit) | Button disabled, `Loader2` icon from lucide-react with `animate-spin`, text "Anmelden" stays |
| Success | `router.replace('/portal')` βÄî no success state rendered on login page |

### Portal Tiles

| State | Visual |
|-------|--------|
| Default | `Card`, white background, `border` border, `rounded-lg`, label centered |
| Hover | `bg-muted` background, 100ms transition, cursor pointer |
| Focus (keyboard) | `ring-2 ring-primary ring-offset-2` visible focus ring |
| Active (click) | `scale-95`, 100ms transition |

### Route Behavior

| Scenario | Behavior |
|----------|----------|
| Unauthenticated βÜ£ `/portal` | Middleware redirects to `/login` |
| Unauthenticated βÜ£ `/flaechen` | Middleware redirects to `/login` |
| Authenticated βÜ£ `/` | Server Component `redirect('/portal')` |
| Authenticated βÜ£ `/login` | Renders normally in Phase 1 (redirect deferred to Phase 3) |

## Layout Contract

### Login Page (`/login`)

- Full viewport height, vertically and horizontally centered (`min-h-screen flex items-center justify-center`)
- Page background: `bg-muted`
- Login card: `Card`, `w-full max-w-sm`, `p-8`, `shadow-sm`
- Title "Anmeldung": 20px/600, `mb-6`
- Form fields stacked, gap `md` (16px)
- Submit button: full width, shadcn `Button` default variant
- Error message: below submit button, `text-destructive`, 14px/400

### Portal Page (`/portal`)

- Page background: `bg-muted`
- Header bar: `bg-background`, `border-b`, height 56px, app name "Campus Platform" left-aligned, 16px/600
- Content area: `max-w-5xl mx-auto px-6 py-8`
- Section heading "Module": 20px/600, `mb-6`
- Tile grid: `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-6`
- Tile: `Card` + `CardContent`, min-height 120px, flex column centered
- Tile icon: lucide-react `LayoutDashboard`, 32px, `text-muted-foreground`
- Tile label: 14px/500, `mt-3`, `text-foreground`
