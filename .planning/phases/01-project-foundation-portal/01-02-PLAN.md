5. `handleSubmit` function (async, on form `onSubmit`):
   ```typescript
   async function handleSubmit(e: React.FormEvent<HTMLFormElement>) {
     e.preventDefault()
     setError(null)
     setLoading(true)

     try {
       const formData = new FormData(e.currentTarget)

       const res = await fetch('/api/users/login', {
         method: 'POST',
         credentials: 'include',
         headers: { 'Content-Type': 'application/json' },
         body: JSON.stringify({
           email: formData.get('email'),
           password: formData.get('password'),
         }),
       })

       if (res.ok) {
         router.replace('/portal')
       } else {
         const data = await res.json()
         setError(data.errors?.[0]?.message ?? 'Anmeldung fehlgeschlagen. Bitte versuchen Sie es erneut.')
       }
     } catch {
       setError('Anmeldung fehlgeschlagen. Bitte versuchen Sie es erneut.')
     } finally {
       setLoading(false)
     }
   }
   ```

CRITICAL: `credentials: 'include'` is required on the fetch call. Without it, the browser will not store the `payload-token` cookie and login will appear to succeed but the user will be redirected back to /login.

CRITICAL: Error message for invalid credentials should be the generic German message "Anmeldung fehlgeschlagen. Bitte versuchen Sie es erneut." as a fallback. Payload's own error message may also be shown if available from `data.errors[0].message`.

Do NOT call `getPayload` or `payload.auth()` in this Client Component — those are server-only functions.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Login form | mitigate | POST to Payload's built-in `/api/users/login` which handles bcrypt verification; no custom auth logic |
| T-02-02 | Information disclosure | Error messages | mitigate | Generic error message "Anmeldung fehlgeschlagen" — does not reveal whether email exists or password is wrong |
| T-02-03 | Information disclosure | Cookie handling | mitigate | `credentials: 'include'` ensures same-origin; Payload sets HTTP-only flag on `payload-token` cookie — JavaScript cannot read it |
